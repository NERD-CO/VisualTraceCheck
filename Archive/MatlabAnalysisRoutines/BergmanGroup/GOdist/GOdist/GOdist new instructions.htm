Analysis of Affymetrix®    microarrays data using GOdist software

 

GOdist is a MATLAB® program that analyzes distributions of transcript values associated with specific gene ontology terms. GOdist is an implementation of the “continuous” approach for analysis of GO term expression patterns. Thus, rather than testing the number of “changed” transcripts associated with each category, GOdist considers all transcripts within the category, and compares their distributions to those of either the entire set of measured transcripts, or to those of their parent terms in the ontology. From a biological point of view, the continuous approach allows detection of changes in categories which are not characterized by a larger than expected number of changed genes, but reflect any changes whatsoever in the form of their distributions.  

GOdist also performs calculations based on the “discrete” approach. GOdist was designed to work with data from Affymetrix®    microarrays and Gene Ontology Consortium (GOC) ontologies. You can go to  http://www.geneontology.org/ web site to read more about the ontologies.

The essential output of GOdist is a list of gene ontology terms, and results from a non-parametric statistical test between the distributions of values (e.g. expression levels) of all transcripts associated with these terms, compared to the distribution of all transcripts measured in the experiment. Results of this analysis are saved in excel files. In addition, GOdist can accept as input specific GO terms, and compare the distribution of transcripts associated with them, to their parent term(s). When a specific GO term is analyzed, a more rigorous analysis is conducted, employing also a non-parametric ANOVA, a test of variance, and a graphic representation of the distribution under comparison. Also, GOdist can get the list of probes, and their matching UniGene codes and expression levels associated with a specific GO term and draw a graphic representation of the number of transcripts which are associated with each expression level in a specific term.

Detailed instructions:

Prepare the data files:

       GOdist uses a variety of input files, taken from different sources.

We use data extracted from Affymetrix®   microarray experiment, data from Affymetrix®     web site  and ontologies data from GOC site.

First, create a new folder which will eventually include all the input files ("main directory").  Choose a short experiment name which we will use later for input files name convention. Now we can prepare the input files.

Files from Affymetrix®     results file:

The microarray data file includes information about the specific inputs added to the Affymetrix®     expression data file (*pivot*.xls). The following additional columns will be created: for each treatment rehearsals, take the result columns and compute the mean, standard deviation and coefficient variation column using excel functions. Than, compute the data column by taking log2 of expression ratio (that is, mean null/mean WT columns). Optionally, compute a scores column. The scores will indicate which transcript to select when several contain the same UniGene code.  The score can be, for example, the variability across multiple treatments.  Sort the file according to (ascending) Probe Set ID Save data and scores columns each in a separate excel (.xls) file. The name of each file should start with the experiment name prefix, underscore ("_") and than "data" or "scores", respectively (experiment_data.xls,experiment _scores.xls). Save the mean column of the treatment (Mean Null) column in a file called experiment_mean_treatment.xls and the mean of WT column in a file called experiment_WT.xls

DON’T PANIC, all data files are prepared for you in advance.

 

Now when all data files are prepared, GOdist is ready to run.

  1. Open Matlab.
  2. Browse GOdist program folder as current directory.
  3. Type "GOdist".
  4. GOdist interface appears. If it does not appear, go to file→set path, add GOdist software folder (not the input files folder, only GOdist program files) to the path and click 'save'.  

Obtaining the results involves 4 stages. Note that each of these stages can only be performed after the previous one is completed.

Stage 1: Data preparation:

Browse the main directory (the directory with all input files prepared in the previous stage).

Type your selected experiment name (which was used as the input files prefix) in the Experiment Name text box and press 'Prepare Data' button.

Stage 2: Choose a transcript selection rule:

It is often the case with Affymterix arrays that several oligonucleotides correspond to the same UniGene cluster. Thus, it does not make sense to consider all oligonucleotide probes for further analysis; but which oligonucleotides should be used? GOdist allows several selection criteria. You can choose transcripts according to their data: Median, Mean, Maximal absolute value, or minimal absolute value (for all transcripts with a given UniGene). Alternatively, and if you created scores file, you can choose the rule according to score (highest/lowest) instead according to the data.  After specifying the selection method, click the 'Apply' button to generate the list of transcripts which will be used for further analysis.

Stage 3: Select ontology to analyze:  

Choose Function for molecular function categories, Process for biological process categories or Component for cellular component categories.

Stage 4: Analyze the data:

Specify change criteria (for the discrete approach) and analyze all terms:

 

  1. Select threshold method (fixed/fraction changed).  The threshold is used by the discrete approach to test for overrepresentation of changed genes in each category (it is irrelevant for the continuous approach). Two types of threshold can be set: fixed (absolute) values, above which a transcript is considered as changed, or fraction changed, in which a certain percentage in the upper tail and lower tail of the distribution are considered as changed. If fixed threshold was chosen, enter a number within data (expression log ratios) column range (or leave 1, which corresponds to 2-fold change, as the default). Otherwise, enter the percentage (between 0-100) in the Fraction text box (or leave the default as 5%).
  2. Select P-value: each of the two statistic tests employed on the data reports a P-value for each tail, for any given category of the selected ontology. We wish to view only the categories with a specified P-value. For example, to view only categories including statistically significant differences with P-value lower than 0.05, enter the P-value 0.05 in the Range box. The default P-value (1) will show all the ontology categories in the results file.
  3. Press Analyze.

 

 

 

When the "Done!" message box appears, it indicates that the result file was loaded into the main data directory (as specified in stage 1). The name of the excel output file will be either: experiment_MF_GO_analysis.xls, experiment_BP_GO_analysis.xls, or experiment_CC_GO_analysis.xls, depending on the ontology selected in stage 3. Just a small reminder: MF stands for Molecular Function, BP stands for Biological Process and CC stands for Cellular Component.  

An example of a result file for all terms analysis:

Each row describes one GO term (for which there was at least one transcript in the dataset). The columns contain the following information:

1. Name of GO term.

2. GO ID of term.

3. P-value for the one tailed Kolmogorov-Smirnov test, testing the hypothesis that the tested distribution contains larger values than the global distribution.

4. P-value for the Fisher Exact test employing the hypergeometric distribution (i.e. the Discrete method), testing the hypothesis that the tested distribution contains a larger than expected number of increased genes (as defined in stage 3).

5. P-value for the one tailed Kolmogorov-Smirnov test, testing the hypothesis that the tested distribution contains smaller values than the global distribution.

6. P-value for the Fisher Exact test employing the hypergeometric distribution (i.e. the Discrete method), testing the hypothesis that the tested distribution contains a larger than expected number of decreased genes (as defined in stage 3).

7. Number of transcripts associated with the GO term (after selection as in stage 2).

8. A number indicating the reliability of the KS test. The p-value is most reliable when this value is larger than 4.

The red and blue cells in the figure above were created manually using the “conditional formatting” option of excel, with value <=0.05 criteria (alternatively, choose P-value as 0.05 to get only the colored rows in the results file).

 

Analyze specific term:

 

In order to take a closer look at specific terms, go to Analyze specific term panel.

First, select the ontology of the term to be analyzed.

 

Than, enter the term number in GO ID box.

1.  To generate cumulative density function (CDF) plots of the selected distribution compared to the global distribution and to the term’s parent, click "View CDF → show".

 

 

An example of CDF plot (Changed genes, according to the specified criteria, are shown as vertical lines):

Statistic parameters meaning explanation: high or low KS statistic, together with low KW test P-value (<0.05/<0.001) indicates whether the two distributions differ in their locations. The variance test shows whether the variance of the selected category (phosphorus metabolism) is larger than that of the global distribution (P < 0.001), suggesting that this category is subject to more tightly constrained regulation than the entire set of transcripts.  

2. To generate a histogram of the number of transcripts associated with each expression ratio of a specific GO term, choose the log ratio (2/10) for expression ratios calculation, and press 'Show'.
